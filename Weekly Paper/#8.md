# 📌 YOLO 모델 개념 & mAP 이해 정리

## 🇶 YOLO(You Only Look Once) 모델의 주요 특징과 장점은?

### ✔ YOLO의 핵심 아이디어
YOLO는 "이미지를 **한 번에(Once)** 훑어보고 객체를 찾아낸다"는 구조다.  
기존 R-CNN 계열처럼 후보 영역을 먼저 뽑고 분류하는 두 단계 방식이 아니라,  
**단일 단계(one-stage)** 로 바로 위치 + 클래스 정보를 예측한다.

> 한 장의 이미지를 격자(Grid)로 나누고,  
> 각 격자가 “내 구역에 객체 있어요?” 하고 동시에 예측하는 방식.

### ✔ YOLO의 주요 특징

- **One-stage Detector**  
  → 빠른 속도에 초점을 둔 구조  
- **End-to-End 구조**  
  → 이미지 입력 → bbox + 클래스 출력까지 단일 네트워크  
- **Real-time Detection**  
  → FPS가 높아 실시간 영상 처리에도 사용 가능  
- **전역적 시야(Global Context)**  
  → 이미지를 전체적으로 보는 구조라 배경·맥락을 잘 이해함  
- **Bounding Box + Class 동시 예측**  
  → 위치와 카테고리를 한 번에 처리  
- **다양한 모델 크기 제공(YOLOv3~v8, nano~xlarge 등)**  
  → 빠르게 할지, 정확도를 챙길지 선택 가능

### ✔ YOLO의 장점

- **매우 빠르다**  
  실시간 카메라·영상에서도 안정적으로 동작  
- **다양한 객체가 함께 등장해도 잘 잡는다**  
  전체 그림을 보며 예측하기 때문  
- **구조가 단순해 적용·학습이 쉽다**  
  전이학습(Transfer Learning)도 수월  
- **경량 모델이 많아 모바일·엣지에서도 활용 가능**  

요약하자면, YOLO는  
“빠르고 정확하며, 실전에서 바로 쓸 수 있는 모델”  
이다.

## 🇶 mAP(mean Average Precision)의 개념은?  
### 객체 인식에서 어떻게 활용될까?

### ✔ mAP란?
mAP는 **객체 탐지 모델의 전반적인 성능을 수치로 요약한 척도**다.  
Precision(정밀도)와 Recall(재현율)을 조합해 만든 AP(Average Precision)를  
모든 클래스에 대해 평균 낸 값이 바로 **mAP**.

> 즉, "모델이 대상을 얼마나 정확하게 찾고,  
> 제대로 찾은 것인지"를 측정하는 종합 점수.

### ✔ AP는 어떻게 계산될까?
- IoU(Intersection over Union) 기준을 정한다 (예: IoU 0.5)  
- Precision-Recall Curve를 그린다  
- 그 곡선 아래 넓이(Area Under Curve)를 AP로 사용한다  

AP는 결국,  
“이 모델이 정답을 얼마나 정확하게, 안정적으로 찾아내는가?”  
를 표현한 값이다.

### ✔ mAP는?

- **mAP@0.5**  
  IoU 0.5 이상인 경우를 기준으로 측정  
- **mAP@0.5:0.95**  
  IoU 0.5부터 0.95까지 0.05 간격으로 평균  
  → 더 엄격함  
  → COCO 대회에서 주로 쓰는 기준  

### ✔ 객체 인식에서 mAP가 중요한 이유

- 서로 다른 모델의 성능을 **공정하게 비교**할 수 있음  
- 클래스 간 불균형이 있어도 전체 성능을 **대표적으로 요약**  
- Threshold(임계값) 변화에도 안정적으로 성능을 반영  
- 학습 중 model checkpoint 선택 기준으로 사용  

요약하자면, mAP는  
“이 모델이 진짜로 잘 찾는지, 수학적 근거로 보여주는 점수”  
라고 보면 된다.
